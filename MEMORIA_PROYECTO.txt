1. OBJETIVO DEL PROYECTO
------------------------

Construir un asistente experto de IA generativa que responda preguntas sobre analitica de futbol profesional usando:

- Google Gemini como LLM.
- Gemini Embeddings para embeddings.
- ChromaDB como base vectorial.
- LangGraph como framework de agente.
- Jupyter Notebook como entorno principal.
- pandas para consultar datos tabulares del CSV.

El proyecto debe cumplir el enunciado:

- Base de conocimiento vectorial con minimo 3 documentos o equivalente a unas 20 paginas.
- ChromaDB + Gemini Embeddings.
- Respuestas usando RAG + Gemini.
- System prompt personalizado y justificado.
- Agente con LangGraph.
- Memoria conversacional.
- Celda de interaccion en notebook.
- Minimo 5 preguntas de ejemplo con respuestas documentadas.


2. DOMINIO ELEGIDO
------------------

Dominio final:

Asistente experto en analitica de futbol profesional, especializado en scouting, rendimiento de jugadores y metricas avanzadas.

El asistente debe ayudar a:

- Explicar metricas avanzadas como xG, xAG, PrgC, SCA, PSxG, PSxG+/-.
- Interpretar perfiles de jugadores.
- Consultar rankings y filtros del dataset de jugadores.
- Combinar explicaciones conceptuales con datos tabulares.


3. ARQUITECTURA DECIDIDA
------------------------

Se eligio una arquitectura hibrida y simple:

A) ChromaDB / RAG:

- Se usa solo para conocimiento conceptual procedente de PDFs de metricas.
- No se indexa todo el CSV en ChromaDB.
- ChromaDB ayuda a responder preguntas conceptuales sobre metricas y buenas practicas de analisis.

B) pandas / CSV:

- Se usa para consultar todos los jugadores del dataset completo.
- Es mejor para rankings, filtros exactos y comparaciones numericas.
- Ejemplos:
  - top jugadores por xG
  - buscar jugador por nombre
  - comparar jugadores
  - filtrar por posicion o minutos

C) Gemini:

- Recibe el contexto recuperado desde ChromaDB.
- Recibe resultados calculados con pandas cuando aplique.
- Redacta la respuesta final en espanol, con tono claro y profesional.

Motivo de la decision:

ChromaDB no es ideal para consultas exactas del tipo xG > 15, Min >= 1000 o top 10 por xAG. pandas es mas fiable para datos numericos. ChromaDB se reserva para RAG conceptual.

2. .env se habia anadido a Git:
   - Se quito con git rm --cached .env.

3. langchain_text_splitters no instalado:
   - Solucion: instalar langchain-text-splitters.
   - Notebook usa try/except para compatibilidad.

4. langchain_community no instalado:
   - Solucion: instalar langchain-community.

5. Modelo embeddings text-embedding-004 no encontrado:
   - Se cambio a gemini-embedding-001.

6. Modelo embedding-001 no encontrado:
   - Tambien se descarto.

7. Cuota Gemini 429 RESOURCE_EXHAUSTED:
   - Se redujo el numero de documentos.
   - Luego se simplifico la arquitectura para indexar solo PDFs, no jugadores.

8. Error NameError: SYSTEM_PROMPT not defined:
   - Causa: ejecutar celda RAG sin ejecutar antes la celda del system prompt.
   - Solucion: ejecutar secciones en orden.

9. Error NameError: shutil not defined:
   - Causa: celda de indexacion ejecutada sin imports previos.
   - Se anadio import local donde hacia falta.
